# econ-insta 피드 개편 — 설계 문서 (디자인 + 콘텐츠)

- **날짜**: 2026-07-16
- **상태**: 승인 대기(스펙 리뷰)
- **범위**: 카드 **디자인 시스템**과 **콘텐츠 구조** 총체 개편. 둘 다 "안 좋다"는 평이 반복됨.
- **비중**: **디자인 8 : 콘텐츠 2.** 디자인이 주인공. 콘텐츠는 어텐션을 돕는 조연.
- **북극성**: "유익한 정보 제공"이 아니라 **"사람을 끌어모으는 계정"**. 성공 지표는 정보의
  깊이가 아니라 **어텐션** — 스크롤을 멈추게 하고(표지), 끝까지 넘기게 하고, 팔로우·저장·공유되는 것.
- **관련 코드**: `econ_insta/renderer.py`(디자인 핵심), `summarizer.py`(콘텐츠 핵심),
  `collector.py`, `reels.py`, `blog_brief.py`, `stock_brief.py`, `ai_brief.py`
- **관련 시안**: `design/covers/cover-{a..e}.html`

---

## 1. 배경 / 문제

인스타 피드가 **디자인·콘텐츠 양쪽 모두** "총체적으로 안 좋다"는 평을 반복해서 받고 있다.

**디자인** — 네 갈래 모두: ① 휑하고 아마추어 같다 ② 스크롤에서 눈길을 못 끈다
③ 촛스러운 감각이 없다 ④ 카드·시리즈 간 통일감이 없다.

**콘텐츠** — 두 갈래: ⑤ **분산** — 표지는 이슈 A를 내걸었는데 뒤 카드들은 무관한
B·C·D를 한 장씩 나열해 "딥다이브"가 아니라 "오늘의 뉴스 모음"이 된다. ⑥ **얕음** —
각 카드가 너무 짧아(2~3문장) "그래서 왜 중요한가"가 없다.

부분 수정이 아니라 **디자인 시스템과 콘텐츠 구조를 함께 다시 세우는 작업**이다.

## 2. 근본 원인 (실제 코드·출력 확인)

### 2.1 디자인
렌더러에는 이미 `Theme` 토큰 시스템이 있고 기본값 `DARK_AMBER`는 네이비+골드 다크
테마다. 색 체계 자체는 문제가 아니었다. 안 좋아 보인 진짜 원인:

1. **폰트 폴백** — `assets/fonts/`가 비어 있어 Pretendard 후보를 건너뛰고 **맑은 고딕**
   (윈도우 기본 UI 폰트)으로 폴백된다. "기본 템플릿 느낌"의 절반이 여기서 온다.
2. **깊이 없는 배경** — 테마 배경이 단색뿐. 그라디언트·질감·글로우가 전무.
3. **소심한 타입 스케일** — 표지 84 / 카드 제목 58 / 번호 64. 볼드 에디토리얼에 못 미친다.
4. **지표 카드가 텍스트 리스트** — `render_indicators`는 8개 지표를 글자 목록으로만 나열.
   시각화(차트)가 없다. (기존의 멋진 면적그래프는 `stock_brief`의 별개 코드였다.)
5. **모듈마다 테마 제각각** — AI=페이퍼, 마켓=크림 등으로 시리즈 통일감이 무너진다.

### 2.2 콘텐츠 (`summarizer.py`)
데일리 브리핑이 **모음집 포맷**으로 설계돼 있다. 이게 분산·얕음의 구조적 원인:

6. **카드 1장 = 서로 다른 기사 1건** — 프롬프트가 *"카드는 3~5장, 파급력 큰 것부터
   고르라"*(summarizer.py:60)고 지시. 표지 `headline`은 뒤 카드들과 아무 연결이 없다.
   → 설계상 딥다이브가 불가능하고 "오늘 뉴스 3~5건 모음"만 나온다.
7. **얕은 본문 상한** — `CARD_BODY_MAX = 120`자·"2~3문장"(summarizer.py:33,63). 각 카드가
   독립된 다른 기사라 "배경 → 무슨 일 → 시장 반응 → 왜 중요 → 앞으로"로 파고들 여지가 없다.

> 참고: `stock_brief`·`blog_brief`는 이미 단일 주제형이고, **데일리 브리핑(`summarizer.py`)이
> 유일한 '모음집' 포맷**이다. `ai_brief`는 의도적 모음집("카드 하나가 사건 하나") — §10에서 별도 논의.

→ 결론: 디자인은 **기존 `Theme` 토큰 위에** 폰트·프리미티브·스케일을 얹고,
콘텐츠는 **데일리 브리핑을 단일 이슈 후크형으로** 재설계한다. 발행 계층은 불변.

## 3. 목표 / 비목표

**목표 (우선순위 = 어텐션)**
- (디자인 — 주력, 8) 볼드 에디토리얼(잡지 표지) 감성의 통일된 카드 시스템.
  - **표지가 최우선 성장 레버.** 인스타 피드·탐색 탭에 뜨는 첫 이미지가 스크롤을 멈추게
    해야 한다 — 초대형 타이포, 강한 대비, 한 방에 읽히는 훅.
  - 메인 = **다크 프리미엄**(네이비 그라디언트 + 골드). 스크롤 스토퍼 = **C 컬러 변주**(버밀리언 풀블리드).
  - **강한 시각 아이덴티티 = 팔로우를 부르는 브랜드.** 데일리·블로그·종목·AI·릴스 전 시리즈가
    같은 토큰·프리미티브를 공유해 한눈에 "이 계정"임을 알아보게 한다.
- (콘텐츠 — 조연, 2) 데일리 브리핑을 **단일 이슈 후크형**으로 전환.
  - 하루 = 가장 화제성 큰 이슈 **1건**. 표지 훅 + 본문 3~4장이 그 하나를 끝까지 넘기게 만든다.
  - 목적은 정보의 깊이가 아니라 **끝까지 보게 하는 서사**(호기심→전개→마무리). 과한 심층·수치 나열 금지.
  - 나머지 그날 뉴스는 **버린다**(모음집 카드 없음 — 분산은 어텐션의 적). 지표 카드는 유지.
- 수집·팩트체크·호스팅·발행 로직은 **불변**. 바꾸는 건 요약 설계와 렌더 계층.

**비목표**
- 발행 자동화(GH Actions cron) 연결 — 별개 작업(메모리의 "남은 일"). 이번 스코프 아님.
- 수집기(`collector.py`) 매체·필터 변경 — 입력은 그대로. 단 지표 시계열 필드는 추가(§5.5).
- `ai_brief`·`blog_brief` 콘텐츠 구조 변경 — §10에서 결정(기본은 이번 스코프 밖).

> 주의: 이전 스펙의 비목표였던 "인물 라이브러리·배경 소스 그대로 재사용"은 **철회**한다.
> 표지가 최대 성장 레버이고 라이선스 제약이 풀렸으므로 **이미지 소싱(`backgrounds.py`) 확장은
> 이번 스코프의 핵심**이다(§5.4).

## 4. 콘텐츠 개편 — 단일 이슈 후크형 (`summarizer.py`)

> 콘텐츠는 조연(비중 2). 목표는 **정보 깊이가 아니라 어텐션** — 표지에서 멈추게 하고
> 끝까지 넘기게 하는 것. 과하게 심층 설계하지 않는다. 단일 이슈 집중은 "깊이" 때문이
> 아니라 **분산이 어텐션을 죽이기** 때문에 채택한다.

### 4.1 이슈 선정 — 인기도 신호로 판별 (LLM 판단만으로는 부족)
"큰 이슈"를 **LLM의 편집 판단**에만 맡기지 않는다. 어텐션이 북극성이므로 **실제 조회수·화제성
신호**로 후보를 좁힌 뒤, 그 안에서 모델이 고른다. 신호(구현 시 가중 결합):
- **크로스소스 빈도**(무료·이미 있음): 여러 매체가 동시에 다룬 사건일수록 큼. 수집물에서 계산.
- **'많이 본 뉴스' 랭킹**: 네이버/다음 경제 인기기사, 연합 인기기사 = 진짜 독자 조회수.
  HTML 스크래핑(RSS엔 없음), 취약할 수 있어 실패해도 발행을 막지 않게(다른 신호로 폴백).
- **트렌딩 검색어**: 네이버 데이터랩(공식 API)·구글 트렌드 = 실시간 대중 관심. 키/레이트리밋.
- 신호가 전부 실패하면 **크로스소스 빈도 → LLM 판단** 순으로 우아하게 저하.

**소스 확대**: 현 5개 매체(연합·한경·매경·WSJ·이코노미스트)에 국내(조선비즈·이데일리·서울경제·
연합인포맥스)·해외(Reuters·Bloomberg·FT·CNBC) 추가. '많이 본 뉴스' 랭킹은 **소스이자 신호**.

- 같은 이슈를 다룬 여러 기사를 **하나로 묶어** 서사의 재료로 삼는다.
  (기존엔 기사 1건→카드 1장. 이제 기사 N건→이슈 1개→카드 N장.)
- **재료 부족 대비**: 한 이슈로 채울 근거가 부족하면(조용한 날) 카드 수를 줄이거나
  다른 이슈로 폴백. 억지로 늘려 추측하지 않는다(팩트체크 원칙).

**발행 편수 (초기 성장기, 사용자 결정 2026-07-16): 하루 3건.** 인기도 신호 상위 **3개 이슈**를
각각 **독립된 단일 이슈 캐러셀**로 발행(기존 1일 1건에서 확대). 이슈끼리 서로 다른 사건이어야
하며(중복 금지), 각 게시물은 표지+본문 3~4장(+지표는 하루 1회분을 첫 게시물에 붙이거나 공유).
인스타 24h 롤링 100건 한도 내라 문제 없음. 초기 노출·팔로워 확보용이고 추후 조정 가능.

### 4.2 카드 서사 구조 (훅 → 전개 → 마무리)
표지가 **훅**, 본문 3~4장이 궁금증을 풀며 끝까지 넘기게 만든다. (엄격 고정 아님, 권장 골격)

| 순서 | 역할 | 내용 |
|---|---|---|
| 표지 | **훅** | 리드 이슈를 스크롤 스토퍼 한 문장으로. 궁금하게 만든다. |
| 1 | 무슨 일 | 핵심 사실. 무엇이·얼마나. |
| 2 | 왜 / 배경 | 맥락. 왜 이런 일이. |
| 3 | 반응 / 파장 | 시장·업계가 어떻게 반응했나. |
| 4 | 그래서 / 앞으로 | 마무리 한 방. 다음 관전 포인트. |
| (지표) | — | 8개 지표 카드(§5.5). 서사와 별개. |

- **분량**: `CARD_BODY_MAX`를 120 → **약 160자**로 소폭 상향(2~3문장, 읽는 부담 없이). 상한은
  새 타입 스케일(본문 46px, 행간 1.5)에서 세로 중앙 밴드에 들어가는 선으로 구현 시 실측 조정.
  *깊이보다 리듬* — 각 카드는 짧고 다음이 궁금하게.
- 표지 `headline`은 리드 이슈 그 자체 → 표지-본문 주제 일치가 구조적으로 보장된다.
  헤드라인은 **훅 카피**로 뽑는다(밋밋한 요약 금지). 단 팩트체크 규칙(숫자 금지 등)은 준수.

### 4.3 스키마 / 검증 변경
- `SCHEMA`에 카드별 `role`(권장 국면 라벨) 추가. `cards`는 순서가 서사 순서.
- `source`는 이슈를 묶으며 **복수 매체**가 될 수 있음 → 카드 푸터는 주요 출처를 표기
  (다중 출처 표기 규칙은 구현 시 확정: 대표 1곳 또는 `A·B`).
- `MIN_CARDS`/`MAX_CARDS`(3~5) 유지. `_validate` 길이 상한을 새 `CARD_BODY_MAX`로.
- **팩트체크 전부 유지**: headline·indicator_note 숫자 금지, 본문 수치=자료 내 값,
  원/달러 방향(`wrong_won_direction`), 근거 없는 수치(`unsupported_amounts`), 1회 재생성.
- 프롬프트에 **단일 이슈 규칙**을 명시(안 넣으면 다시 모음집으로 회귀 — ai_brief에서 실증됨).

### 4.4 안전 / 회귀
- `Briefing`/`Card` 데이터 형태 변경은 렌더·발행 호출부에 파급 → 함께 수정.
- 조용한 날(단일 이슈 재료 부족)에도 발행이 깨지지 않도록 폴백 경로를 테스트로 고정.

## 5. 디자인 개편 — 디자인 토큰 & 레이아웃

### 5.1 폰트 (최우선)
- **Pretendard 5종 번들**: `assets/fonts/Pretendard-{Black,ExtraBold,Bold,SemiBold,Regular}.otf`.
- 라이선스: **SIL Open Font License (OFL)** — 상업 이용·임베드·저장소 커밋·CI 사용 자유.
  (저장소는 이미 PUBLIC이므로 임베드 가능 라이선스여야 한다 — OFL 충족.)
- `FontSet`을 5단 굵기로 확장. 굵기가 없으면 한 단계 낮은 굵기로 우아하게 폴백
  (Pretendard 없는 환경 → 맑은 고딕/나눔은 Regular·Bold만 있으므로 Black→Bold).
- 기존 `ECON_INSTA_FONT*` 오버라이드·후보 탐색 경로는 유지. CI의 `fonts-nanum` 스텝은
  더 이상 필수 아님(번들 폰트가 우선). 단 폴백 경로로 남겨둔다.

### 5.2 팔레트 / 테마
새 기본 테마 **`DARK_PREMIUM`** (기존 `DARK_AMBER`를 정련):
- 배경: 세로 그라디언트 `#0B0E16 → #141020` (네이비→플럼, 미세하게).
- 표지 글로우: 골드 `#F2C54E` 라디얼(저알파), 좌하단.
- 옅은 그리드 질감: 흰색 108px 격자, alpha≈10.
- 액센트(골드): `#F2C54E`. 전경 `#F5F6FA`, 뮤트 `#8B93A7`, 룰 `#2A303E`, 본문 `#CED2DE`.
- 등락: **상승 빨강 `#F05A60` · 하락 파랑 `#5B8DEF`** (다크 배경용, 관행 유지).

**C 컬러 변주** 시그니처: 버밀리언 `#F04E2A`, 그 위 잉크 `#16110E`/흰색.

기존 테마 `PAPER`/`MIDNIGHT`/`MONO`는 **보존**(회귀·선택지). `DEFAULT_THEME`만 교체.

### 5.3 타입 스케일 (1080×1350, margin 92)
| 용도 | 굵기 | 크기 |
|---|---|---|
| 표지 헤드라인 | Black | 104 (2줄 기준, 넘치면 축소) |
| 표지 키커 | Bold | 38 |
| 카드 번호 | Black | 120 |
| 카드 제목 | ExtraBold | 74 |
| 본문 | Regular | 46 (행간 1.5) |
| 지표 값/변화 | Bold | 스케일 대응 |
| 캡션/출처/페이지 | Regular/SemiBold | 28~34 |

동적 축소 규칙(`_indicator_layout`류)은 유지 — 긴 헤드라인·8개 지표·긴 본문에서 넘치지 않게.

### 5.4 표지 이미지 전략 (최대 성장 레버)

**지배 원칙: 이미지 최대 활용.** 거의 모든 표지에 **실제 이미지**를 넣어 사람을 끌어당긴다.
인물 사진이 있으면 무조건 그 인물 사진. 인물이 없어도 로고·극적 실사로 이미지를 채운다.
순수 타이포/그래픽은 **정말 쓸 이미지가 없을 때의 최후 수단**이지 기본값이 아니다.
단, **사물 설명 사진(웨이퍼·건물 정면 등)은 금지** — 이미지를 쓰되 이목을 끄는 이미지여야 한다.

> **라이선스 정책 변경 (사용자 결정 2026-07-16, 명시적):** 기존의 "뉴스·게티·로이터 사진
> 금지(DMCA 누적→계정 정지)" 제약을 **해제**한다. 사용자가 *"신고당할 일 없다, 내가 책임진다"*
> 로 리스크를 감수. → 표지 이미지 소스가 PD/CC에 국한되지 않고 **실사 사진 전면 허용**된다.
> (이는 메모리에 기록된 프로젝트 결정을 사용자가 뒤집은 것 — 승인 후 메모리 반영.)

우선순위 (이목 집중 순):

1. **인물 얼굴 우선** — 이슈 중심에 유명 인물이 있으면 그 얼굴을 **크게 전면 배치**한다.
   얼굴은 인스타에서 가장 강력한 스크롤 스토퍼다.
   - 라이선스 제약이 풀렸으므로 **PD/CC 초상 라이브러리에 없던 인물(민간 CEO·이창용 등
     한국은행·국내 인물)도 실제 인물 사진으로 커버 가능** — 전략 A의 최대 공백이 메워진다.
   - 기존 `backgrounds.py`의 **좌우 나란히 콜라주(뉴스 썸네일식)를 폐기**하고, 단일 인물이면
     얼굴을 화면을 채우는 크기로(블리드), 2인이면 대비 구도(분할)로.
   - 감정·표정이 강한 컷을 고른다(놀람·긴장·환호). 밋밋한 증명사진식은 피한다.
   - 다크 프리미엄 스크림/그라디언트로 눌러 하단에 훅 헤드라인을 얹는다(`_photo_shade` 재사용).
2. **기업 로고 (기업 이슈)** — 이슈가 특정 기업 중심이고 얼굴이 마땅치 않으면(또는 얼굴보다
   브랜드가 셀 때) **그 기업 로고를 히어로로**. 삼성·엔비디아·SK하이닉스·애플 등 알아보는
   로고는 그 자체로 즉각적인 스크롤 스토퍼다. 시그니처 색 위에 로고 크게, 또는 로고+데이터/
   로고+얼굴 조합. 위키미디어 공용 브랜드 로고(SVG/PNG) 등에서 소싱.
   - 로고는 상표(트레이드마크)라 저작권과 별개 이슈지만, 라이선스 리스크는 사용자 소유 결정(위)에 포함.
3. **극적인 실사 사진** — 인물도 기업도 주인공이 아닌 이슈(거시·금리·인플레 등)도 **순수 그래픽으로
   가지 말고** 사건 현장·상징 장면의 **드라마틱한 실사**로 이미지를 채운다(사물 설명 사진과 다름 —
   긴장·규모·움직임이 있는 컷: 거래소 객장·시위·공장 라인·도시 스카이라인 등). 실사 개방으로 가능.
4. **대담한 그래픽 (최후의 수단)** — 위 셋으로 **쓸 이미지를 도저히 못 구할 때만**: 초대형 타이포 +
   시그니처 색 + **데이터를 히어로로**(폭락 차트를 표지 주인공으로). C 컬러 변주(표지 C)가 이 경로다.
   기본값이 아니라 폴백이다.

**이미지 소싱 파이프라인 (핵심 과제)**: 거의 모든 표지에 이미지를 대려면 소싱이 안정적이어야 한다.
`backgrounds.py`를 확장/재편 — 인물 라이브러리 **얼굴 커버리지 대폭 확대**(케빈 워시 등 현직 인물 포함,
§10) + 기업 로고 소스 + 폭넓은 실사 이미지 검색(라이선스 개방으로 뉴스·웹 이미지 활용 가능).
품질 관리를 위해 자동 1등 선택 금지(메모리: 팬아트·오식별 사례) — 후보 대조 후 선택 흐름 유지.

> 표지는 **인물 → 로고 → 극적 실사**로 최대한 이미지를 채운다. 순수 그래픽은 최후의 수단이라
> 표지가 비거나 밋밋해지는 일은 없다.

### 5.5 카드별 레이아웃

**표지 A (다크 프리미엄, 기본)**
- 그라디언트 배경 + 옅은 그리드 + 좌하단 골드 글로우.
- 좌상단: **키커 pill**(골드 아웃라인 라운드) + 날짜(뮤트).
- 하단 앵커: 골드 **액센트 바** + 초대형 헤드라인(Black, 하단 정렬).
- 푸터: `@mansuki101`(좌) · `넘겨서 확인하세요 →`(우, 뮤트).
- **사진 표지 경로 유지**: 배경 이미지가 오면 기존 스크림(`_photo_shade`)로 눌러
  흰 제목을 하단에. 이 로직은 그대로 재사용.

**표지 C 변주 (풀블리드 컬러)**
- `render_cover(..., variant="color")` 플래그로 켠다. 기본은 `"dark"`.
- 풀블리드 시그니처색 + 상단 라벨줄(영문 키커 + 날짜 + 흰 룰) + 중앙 초대형 잉크 타이포
  + 좌하단 큰 번호 워터마크 + 우하단 CTA.
- 용도: 큰 이슈·단일 주제일 때 수동으로 선택(당분간 호출부 인자). 매일 X.

**본문 카드**
- 좌측 골드 세로 바(풀하이트, 폭 12).
- 상단: 큰 번호(Black 120, 골드) + 우측 `n / total`(뮤트).
- 제목(ExtraBold) + 디바이더 룰 + 본문(Regular, 행간 1.5) — 머리말/꼬리말 사이 **세로 중앙정렬**(기존 규칙 유지).
- 하단: `출처 · {source}`.

**지표 카드 (리스트 + 미니 스파크라인)**
- 8개 지표를 **행 리스트로 유지**(한눈에 보는 정보량 보존).
- 각 행: `이름` · (우측) `현재값` `등락%` + **행 오른쪽/아래에 미니 스파크라인**
  (최근 시계열 축소 라인, 등락색으로 채움).
- 넘침 방지: 기존 `_SCALES` 단계 축소 로직을 스파크라인 높이까지 포함해 확장.
- **데이터 요구**: 각 지표에 짧은 최근 종가 시계열(예: 최근 20~30개)이 필요.
  - `Quote`(또는 지표 모델)에 `series: list[float] | None` 추가. yfinance에서 채운다
    (`auto_adjust=False` — 배당조정가 금지, 메모리 기록된 함정).
  - **우아한 저하**: series가 없으면 그 행은 스파크라인 없이 값만 렌더(발행을 막지 않는다).
    배경 실패가 발행을 막지 않는 프로젝트 원칙과 동일.

**릴스 (9:16, 1080×1920)**
- 세로 레이아웃(`reels.py`)에 **같은 토큰**(폰트·색·액센트) 적용. 카드 레이아웃을 늘려
  쓸 수 없어 별도지만, 토큰은 공유해 톤을 통일한다.

### 5.6 코드 구조 / 포팅 방법
**새 렌더 프리미티브**(renderer.py 내부, 순수 함수):
- `vgrad(top, bot)` — 세로 그라디언트 캔버스.
- `glow(base, xy, r, color, alpha)` — 가우시안 블러 라디얼 글로우.
- `grid(draw, color, step, alpha)` — 옅은 격자.
- `sparkline(draw, series, box, color)` — 박스 안 축소 라인/면적.
- `kicker_pill(draw, text, ...)` — 라운드 아웃라인 라벨.

**Theme 확장**: `bg`를 단색 대신 `bg_top/bg_bottom`(그라디언트) 지원. 기존 단색 테마는
top==bottom으로 표현(하위호환). `accent_glow`, `variant_signature` 필드 추가(옵션).

**FontSet 확장**: `at(size, weight="regular"|"semibold"|"bold"|"extrabold"|"black")`.
누락 굵기는 인접 굵기로 폴백.

**호출부**: 각 시리즈 모듈(`blog_brief`/`stock_brief`/`ai_brief`/`reels`)이 같은
`DEFAULT_THEME`을 쓰도록 통일. 테마 이름 매핑 테이블(예: ai_brief의 `다크/페이퍼`)을
새 기본으로 정렬.

목업 검증 코드는 스크래치패드(`mock.py`)에 있고, 이를 renderer 프리미티브로 이식한다.

## 9. 안전 / 롤백
- (디자인) 기존 테마 보존 + 기본값만 교체 → `DEFAULT_THEME` 한 줄로 롤백 가능.
- (콘텐츠) 요약 재설계는 프롬프트·스키마 중심 → 이전 버전으로 되돌리기 쉬움. 발행/호스팅/토큰 로직 무변경.
- 스파크라인 데이터·단일 이슈 재료 실패는 우아하게 저하(발행 차단 없음).
- **라이선스 리스크는 사용자가 명시적으로 소유(2026-07-16)** — 실사 사진 사용에 따른 저작권/
  DMCA 리스크는 사용자 책임 결정. 렌더러/발행 안전성과는 별개의 사업적 판단.

## 10. 미해결 / 후속
- **`ai_brief` 콘텐츠 구조**: 지금은 의도적 모음집("카드 하나가 사건 하나"). 데일리와 같은
  단일 이슈 후크형으로 통일할지 여부 — 리뷰에서 결정. 기본값은 현행 유지(이번 스코프 밖).
- C 변주 자동 선택(모델이 큰 이슈 판단) — 이번엔 수동 인자, 후속에서 검토.
- 지표 스파크라인용 시계열 수집이 발행 시간을 늘릴 수 있음 — 8종목 배치 조회로 최소화.
- **인물 라이브러리 최신화**: 현직 인물이 빠져 있다 — 예: **현 연준 의장은 케빈 워시(Kevin Warsh),
  파월 아님**(2026 기준). 얼굴 표지를 쓰려면 워시 등 현직 인물 사진을 소싱해 라이브러리에 추가.
  실사 개방으로 확보 가능. (목업의 파월 얼굴은 데모 플레이스홀더였음.)
- (스코프 밖) GH Actions cron 자동 발행 연결은 별도 작업으로 남음.
