# 바이라인 오염 — 클러스터링이 매체 서명으로 뭉친다

- 관련 스펙: `2026-07-17-collector-quota-design.md` §6 실데이터 검증이 발견, §7이 "관측 전 손대지 말라"고 미뤄둔 항목
- 브랜치: `collector-quota` (Task 4 `5026e8d` 위)
- 날짜: 2026-07-17

## 1. 문제 (실측, 추측 아님)

quota 폐기 후 첫 실데이터 검증(171건)에서 이슈 2위가 **기사 64건짜리 쓰레기 메가클러스터**로 나왔다
(오세훈 증시발언 + 서울시설공단 가이드 + 무신사 협업상품 + 호르무즈 봉쇄 + 임금체불…).
정체 = 그날의 연합뉴스 거의 전부(수집 70건 중 64건).

원인 실측: 붙은 기사들이 시드와 공유하는 핵심어가 `['기자','김준태','서울','연합뉴스']` /
`['기자','연합뉴스']` … **주제어가 하나도 없다.** 연합뉴스 RSS summary가
`(서울=연합뉴스) 김준태 기자 = 본문…`으로 시작하고, `keywords()`가 title+summary를 훑으므로
`{기자, 연합뉴스}`만으로 `min_shared=2`가 충족된다. **같은 매체 기사끼리 주제와 무관하게 무조건 병합.**

`issues.py`의 "시드 핵심어에만 매칭(과합병 방지)"은 시드 핵심어 자체가 바이라인을 담아 무력하다.
1위 이슈(AI 혁명, 매체 3곳)에 광주은행 노조 기사가 붙은 것도 같은 병이다.

**quota 폐기가 만든 회귀가 아니다** — 19건 규모(연합 4건)에선 뭉칠 게 없어 안 보였을 뿐. 규모가 드러냈다.

바이라인 실측 (2026-07-17): 연합만 바이라인이 있다. 매경·WSJ는 본문 직시작, 한경은 summary 0건.
일부 연합 기사(■ 헤드라인 모음)는 바이라인이 없다.

## 2. 사용자 결정 (2026-07-17)

collector-quota 브랜치에서 같이 고친다. **수정 지점 = collector에서 벗긴다** (keywords()-만 방어 기각:
기자 이름은 열거 불가라 스톱워드로 못 막고 — 실측에서 김준태 기자 혼자 여러 건 — 제목만 쓰기는
매경 summary의 진짜 주제어까지 버린다).

## 3. 설계

### 3.1 collector — `strip_byline()`

```python
_BYLINE_RE = re.compile(r"^\([가-힣]+=[^)]+\)\s*[가-힣·\s]{2,30}?(기자|특파원)\s*=\s*")

def strip_byline(text: str) -> str:
    """연합뉴스식 바이라인 접두 '(서울=연합뉴스) 김준태 기자 = '를 벗긴다.

    지역·매체·기자명은 가변이라 구조로 잡는다. 문두 앵커(^)라 본문 중간의
    인용("…라고 김준태 기자가 전했다")은 건드리지 않는다. 바이라인이 없으면 그대로.
    """
    return _BYLINE_RE.sub("", text, count=1)
```

적용 지점 2곳 — **`[:300]` 자르기 전에** 벗긴다(바이라인이 300자 예산을 먹으면 안 됨):
- `collector.py:307` (RSS): `summary=strip_byline(clean_text(_text(item, "description")))[:300]`
- `collector.py:331` (Atom): `summary=strip_byline(clean_text(summary))[:300]`

클러스터링·프롬프트·팩트체크가 보는 텍스트가 같이 깨끗해진다.

### 3.2 issues — `_STOPWORDS` 방어선

`기자`·`특파원`과 매체명(`연합뉴스`·`한국경제`·`매일경제`)을 추가. 바이라인 변형이 정규식을
새더라도 2단어 문턱의 절반을 깎는다. 기자 이름은 못 막는다 — 그래서 3.1이 주 수정이고 이건 방어선이다.

## 4. 테스트

| 테스트 | 잡는 뮤테이션 |
|---|---|
| 바이라인 접두가 벗겨진다 (서울/제주 등 지역 변형 포함) | `strip_byline` 미적용 |
| 특파원 변형도 벗겨진다 | 정규식이 `기자`만 잡음 |
| 바이라인 없는 summary는 그대로 | 과도한 제거 |
| 본문 중간의 "…기자 = " 패턴은 안 벗겨진다 | 앵커(^) 제거 |
| 벗긴 뒤 300자 자르기 (순서) | `[:300]` 뒤에 벗김 |
| `_STOPWORDS`에 기자·매체명 포함 | 방어선 삭제 |

각 뮤테이션을 실제 적용해 FAIL 재현 후 되돌린다 (이 저장소 원칙).

## 5. 실데이터 합격 기준

오늘 피드 재실행에서: ① 64건 메가클러스터 해체(연합 기사가 주제별로 갈라짐),
② 상위 10개 이슈에 바이라인 토큰(`기자`·매체명)만으로 묶인 클러스터가 없음,
③ quota 스펙 §6.3(매체 2곳 이상 ≥ 1개) 여전히 충족.

## 6. 하지 않는 것

`[연합뉴스 이 시각 헤드라인]` 모음(6위)·`[포토]` 캡션(5위) 정형기사 필터 확장 — 바이라인과 별개.
`min_shared`·클러스터링 알고리즘 튜닝 — 바이라인 제거 후 재관측이 먼저다.
